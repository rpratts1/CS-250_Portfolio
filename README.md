##  Agile & Scrum Experience  
During this course, I took on multiple Scrum roles, including:

### **Product Owner**
- Created user stories and acceptance criteria  
- Managed and refined the product backlog  
- Explained the product vision and customer needs  

### **Scrum Master**
- Facilitated Daily Scrum, Sprint Planning, Sprint Review, and Sprint Retrospective  
- Supported the team by removing blockers  
- Encouraged teamwork, transparency, and continuous improvement  

### **Developer**
- Broke down user stories into smaller tasks  
- Helped deliver working software in small increments  
- Practiced estimation techniques such as T-Shirt Sizing  

### **Tester**
- Wrote test ideas based on acceptance criteria  
- Practiced Test-Driven Development (TDD) principles  
- Provided fast feedback to improve software quality  

These experiences helped me understand how Agile teams work together to deliver value.

##  SNHU Travel Project Summary  
The SNHU Travel project allowed me to simulate working on an Agile development team for a travel application.  
Key learning outcomes included:

- Understanding Scrum ceremonies and their purpose  
- Handling interruptions using Agile problem-solving  
- Communicating effectively with teammates  
- Using artifacts like the Definition of Done, user stories, and task boards  
- Comparing Agile and Waterfall to determine which approach fits a project
